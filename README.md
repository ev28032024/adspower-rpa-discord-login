# Инструкция по использованию RPA для автоматической авторизации в Discord.

#### Требования для корректной работы: ```Google Таблица с актуальными данными.```

---
1. **В интерфейсе AdsPower выберите профили, на которых будет осуществляться автоматизация. (Можно скопировать разом все из таблицы и просто вставить в строку)**

<img width="1279" height="510" alt="изображение" src="https://github.com/user-attachments/assets/eafce50e-bb08-45d5-b92e-a5fb5f3bdd09" />

---
2. **Запустите процесс RPA, связанный с логином в Discord: в меню RPA выберите нужный процесс.**

<img width="812" height="91" alt="изображение" src="https://github.com/user-attachments/assets/62d8f7a6-4085-4e4b-b7df-ebe50db7c9db" />

---
3. **Нажмите кнопку запуска и дождитесь выполнения. Посмотреть ход процесса можно в журнале задач («Task Log») AdsPower.**

<img width="768" height="531" alt="изображение" src="https://github.com/user-attachments/assets/4ffb99ba-77c1-4699-b85b-4407c4156ee2" />

---
**Посмотреть логи на предмет успешного или ошибочного выполнения RPA можно в каталоге:** ```~/.config/adspower_global/cwd_global/source/data```

<img width="746" height="612" alt="изображение" src="https://github.com/user-attachments/assets/0bd4eaa4-0147-4a11-886d-70391d6d3fc5" />

---
# Статусы:
**discord_login_<номер_профиля>-<номер_задачи>.txt** - ```Успешно авторизован.```

**discord_not_login_<номер_профиля>-<номер_задачи>.txt** - ```Не удалось авторизоваться.```

**discord_nick_<номер_профиля>-<номер_задачи>.txt** - ```Авторизован, однако никнейм аккаунта не совпадает с никнеймом в таблице.```
